# Train π₀.₅ (pi0.5) on H100 — makermods_pick_book_parallel_grasp

Finetunes [π₀.₅](https://huggingface.co/docs/lerobot/main/en/pi05) on
`srik410/makermods_pick_book_parallel_grasp_merged` (SO-101, single `hand_cam`,
6-DoF, 45 episodes) on a single H100, using a **conda** environment.

> Why finetune and not run base? `lerobot/pi05_base` has never seen the SO-101
> embodiment — running it as-is moves the arm randomly. Finetuning maps its
> general manipulation priors onto your arm. pi0.5 pads your 6-dim action/state
> to 32 internally, so there are **no dimension mismatches** to patch.

SSH into the box and run everything below there:

```bash
ssh ubuntu@<H100_PUBLIC_IP>
```

---

## 1. One-time setup on the H100 (conda)

```bash
# system deps (ffmpeg required for video-backed datasets)
if command -v apt-get >/dev/null 2>&1; then
  sudo apt-get update -y
  sudo apt-get install -y git git-lfs ffmpeg build-essential
fi
git lfs install || true

# install Miniconda if it's not already there
if ! command -v conda >/dev/null 2>&1; then
  wget https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-x86_64.sh -O ~/miniconda.sh
  bash ~/miniconda.sh -b -p ~/miniconda
  source ~/miniconda/etc/profile.d/conda.sh
  conda init bash
fi
source ~/miniconda/etc/profile.d/conda.sh

# code
[ -d lerobot ] || git clone https://github.com/huggingface/lerobot.git
cd lerobot

# accept Anaconda channel Terms of Service (conda blocks env creation otherwise)
conda tos accept --override-channels --channel https://repo.anaconda.com/pkgs/main
conda tos accept --override-channels --channel https://repo.anaconda.com/pkgs/r

# conda env with the Python lerobot needs (>=3.12)
conda create -y -n lerobot python=3.12
conda activate lerobot

# deps: [pi]=paligemma/openpi backbone, [dataset]=HF datasets loader,
#       [training]=accelerate trainer
pip install --upgrade pip
pip install -e ".[pi,dataset,training]"

# sanity
python -c "import numpy, torch; print('torch', torch.__version__, 'cuda', torch.cuda.is_available(), torch.cuda.get_device_name(0))"
which lerobot-train

# auth: needed to pull the dataset (if private) + push the trained policy
hf auth login        # paste your hf_xxx token when prompted
```

## 2. Create the training script

From inside `~/lerobot`:

```bash
cat > train_pi05.sh <<'TRAIN'
#!/usr/bin/env bash
# Finetune pi0.5 on the SO-101 pick-book dataset. Single H100 (80GB).
set -euo pipefail

DATASET="srik410/makermods_pick_book_parallel_grasp_merged"
HF_USER="srik410"
RUN="pi05_book_parallel_grasp"
OUT="outputs/train/${RUN}"

STEPS=10000        # 45 episodes is small; 5k-15k is a sane range
BATCH=32           # train_expert_only fits 32+ on 80GB; drop to 16/8 if OOM
WORKERS=8
SAVE_FREQ=2000
LOG_FREQ=100

# pi0.5 normalizes state/action with QUANTILES (its native, recommended mode).
# Compute + push the quantile stats once. Harmless to re-run; remove after success.
python src/lerobot/scripts/augment_dataset_quantile_stats.py --repo-id="${DATASET}"

# Resume automatically if a checkpoint already exists.
RESUME_ARGS=()
if [ -d "${OUT}/checkpoints/last" ]; then
  echo ">> Resuming ${OUT}"
  RESUME_ARGS=(--config_path="${OUT}/checkpoints/last/pretrained_model/train_config.json" --resume=true)
fi

lerobot-train \
  --dataset.repo_id="${DATASET}" \
  --policy.type=pi05 \
  --policy.pretrained_path=lerobot/pi05_base \
  --policy.device=cuda \
  --policy.dtype=bfloat16 \
  --policy.train_expert_only=true \
  --policy.gradient_checkpointing=true \
  --policy.compile_model=true \
  --batch_size="${BATCH}" \
  --num_workers="${WORKERS}" \
  --steps="${STEPS}" \
  --save_freq="${SAVE_FREQ}" \
  --log_freq="${LOG_FREQ}" \
  --output_dir="${OUT}" \
  --job_name="${RUN}" \
  --policy.push_to_hub=true \
  --policy.repo_id="${HF_USER}/${RUN}" \
  --wandb.enable=false \
  "${RESUME_ARGS[@]}"

echo "Done. Checkpoints in ${OUT}/checkpoints/ (last/ is newest)."
TRAIN
chmod +x train_pi05.sh
```

> **Quantiles vs MEAN_STD:** quantiles is pi0.5's default and the better choice —
> keep the `augment_dataset_quantile_stats.py` step. Only if that step can't run
> (e.g. you can't push to the dataset repo) fall back by deleting that line and
> adding to the `lerobot-train` call:
> `--policy.normalization_mapping='{"ACTION":"MEAN_STD","STATE":"MEAN_STD","VISUAL":"IDENTITY"}'`

## 3. Launch training

```bash
conda activate lerobot
cd ~/lerobot
bash train_pi05.sh 2>&1 | tee ~/pi05_train.log
```

> Runs in the foreground (stops if SSH drops). Keep the session open, or wrap
> with `nohup ... &` if you need it to outlive a disconnect.

## 5. Get the trained policy back (on your Mac)

Auto-pushed to the Hub, so simplest:

```bash
hf download srik410/pi05_book_parallel_grasp --local-dir ./pi05_book_parallel_grasp
```

Or copy the raw checkpoint off the box:

```bash
rsync -avz ubuntu@<H100_PUBLIC_IP>:~/lerobot/outputs/train/pi05_book_parallel_grasp/checkpoints/last/pretrained_model/ \
  ./pi05_book_parallel_grasp/
```

## 6. Run the policy on your SO-101 (from the Mac)

`lerobot-rollout` deploys the policy on the real robot (in this repo
`lerobot-record` is data-collection only). Camera key must match training:

```bash
lerobot-rollout \
  --policy.path=srik410/pi05_book_parallel_grasp \
  --robot.type=so101_follower \
  --robot.port=/dev/ttyACM0 \
  --robot.id=my_so101 \
  --robot.cameras="{ hand_cam: {type: opencv, index_or_path: 0, width: 640, height: 480, fps: 30}}" \
  --task="pick up the book" \
  --duration=60
```

⚠️ Keep a hand on the e-stop the first time — verify motion before trusting it.

## Knobs / troubleshooting

- **OOM on the H100?** Lower `BATCH` (32 → 16 → 8) in `train_pi05.sh`.
- **GPU underused / slow?** Bump `WORKERS` (12–16); video decode is usually the
  bottleneck before GPU compute.
- **Full finetune instead of expert-only?** Set `--policy.train_expert_only=false`
  (more VRAM; risky on only 45 episodes).
- **Want W&B?** Run `wandb login` in step 1 and flip `--wandb.enable=true`.
- **Stop the run:** `Ctrl-C` in the training shell.
- **Tear down the H100** when finished so you stop paying for it.
